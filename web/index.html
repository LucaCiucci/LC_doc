Writing documentation comments

Suppose we have a function we want to documentate: 

int sum(int a, int b);

Brief and Long description

Brief description only

The simplest way is to use a triple / (///) and a brief comment that does not contain a blank line 

/// This is a brief description
int sum(int a, int b);
Anothe way is to use a c-style comment: 

/** or /*!
* This is a brief description
... or ...
* @brief This is a brief description
*/
int sum(int a, int b);
Where the @brief keyword can be omitted.

Adding a long description

You can add a long description by adding a blank line that separates the brief description, the @brief is not striclty necessary 

/**
* This is a brief description
* 
* This is a long description
* Lorem ipsum dolor sit amet consectetur adipisicing elit.
* Vitae sequi voluptas quod iure soluta quo,
* dignissimos neque distinctio dolor earum repellat
* placeat tempora delectus laboriosam deserunt eaque
* at consequuntur et!
*/
int sum(int a, int b);

Parameters

LCDoc supports Doxygen style parameters documentation, it also adds support for bried and long description for each parameter 

/**
* This is a brief description
* 
* This is a long description
*
* @param a param `a` brief description
* @param b param `b` brief description
* 
*     param `b` long description...
* 
* @return sum return brief description
*
*     return long description...
*/
int sum(int a, int b);

Hyperlinks format

Theese are the rules that the lcdoc uses to format hyperlinks:
  • <a href="!something"> will be transformed into <a href="something">, i.e. the ! will be removed and the link is not changed, use this to avoid any other rule application and leave the link "as is"
  • <a href="./something"> this is a relative link, it will be left has-is in the final representation
  • <a href="../something"> this is a parent relative link, it will be left has-is in the final representation
  • <a href="/something"> this is an origin-relative link, it will be transformed into a relative link by default, but you can force it to be an absolute link by using the <a href="!/something"> or by the corresponding option, see the <a href="!something"> rule
  • <a href="//example.com"> is a scheme-relative url, it will be left has-is in the final representation
  • <a href="$something"> will be transformed into <a href="resources-path/something"> or a different path if that particular resurce is different for the current file
  • <a href="$$something"> will be transformed into <a href="resources-path/something"> regardless of the current file overrides
  • <a href="::something"> is a symbol link, it will be transformed into a link to the symbol documentation. The symbol link will be transformed into a link to the symbol documentation, if the symbol is not found, the link will be left as is.
    Note that some symbols have external documentation, for example the C++ standard library, and in that case the link will be transformed into a link to the external documentation. In this case if the there are ambiguities for the usage of different languages, the link will be left as is but you can use the <a href=":::language-identifier::something"> rule to force the link to be transformed into a link to the symbol documentation of the specified language. For example: 
    
@see ::std::vector
@see ::print error if there are multiple languages in the current project
@see :::py::print resolve to the python documentation of the print function
@see :::js::print resolve to the javascript documentation of the print function
See also the MDN <a> reference.

HTML code inside equation

You can also insert HTML elements inside equations usin the \slot command: 
<tex-math>
    R = \frac{\slot{1}}{\slot{second}}

    <code slot="1" style="text-align: left;">hello <code-class>Class</code-class></code>
    <span slot="second" style="color: green;"> second slot</span>
</tex-math>

Numbering

You can add a number to a block equation by simply add one of the following attributes:
  • id
  • n
  • number
Example: 
<tex-math n>
    x^2
</tex-math>
Renders:
x^2

Code Snippets

You can use markdown style code (\` and \`\`\` syntax), or the html <code> tag. If you want to escape the \` character, you can use the \` escape, or the html escape &#96; escape \\ and inline code with \`...\`

Using the <code> tag

You can use the <code> tag to archive the same result, it also gives you more freedom.
Simple <code> tags are transformed into <code class="nohighlight"> to disable highlightjs with plain code snippets. To disable di behaviour, simply add some class or attribute to the tag. For example, (for clarity) type <code highlight> or <code something> or any similar to enable highlightjs automatic language detection and syntax highlighting
Using the <code> tag will trigger highlightjs: 

<pre><code highlight>
<!-- auto language detection -->
</code></pre>

<pre><code class="language-cpp">
int i = 42;
</code></pre>

<pre><code class="nohighlight">
// no syntax hilight here
int i = 42;
</code></pre>
This also works with inline code! 

inline code <code hl><!-- auto language detection --></code>!

<pre><code class="language-cpp">
int i = 42;
</code></pre>
You can manually highlight code in this way: 

<pre><code class="nohighlight">
<code-keyword>class</code-keyword> <code-class>MyClass</code-class>;
</code></pre>
that will be rendered as: 

class MyClass;
